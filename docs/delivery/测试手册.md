# 测试手册

## 1. 目标

本手册用于把 RK3588 离线 TTS 的测试用例从单条冒烟提升为：

- 工程化：测试计划、阈值、输出目录和报告格式统一管理。
- 规模化：支持批量用例、warmup、重复执行、标签筛选和长稳短循环。
- 可视化：自动生成 JSON、CSV、Markdown 和 HTML 仪表盘。

## 2. 测试入口

主入口：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run
```

默认行为：

- 优先读取 config/local/tts_test_plan.json
- 若本地计划不存在，则回退到 config/examples/tts_test_plan.example.json
- 板卡连接参数继续沿用 config/local/board.local.env 和 config/local/delivery.local.env

## 3. 测试计划

推荐做法：

1. 将 [config/examples/tts_test_plan.example.json](config/examples/tts_test_plan.example.json) 复制为 config/local/tts_test_plan.json
2. 按项目阶段维护不同类别的用例：smoke、latency、domain、stability、acceptance
3. 为每个用例设置 repeat、warmup、latency_threshold_ms、rtf_threshold 等约束

支持的核心字段：

- id：稳定的用例标识
- name：报告展示名称
- category：测试分类
- tags：标签，用于筛选批量执行
- sentence：待合成文本
- repeat：有效统计轮次
- warmup：预热轮次
- latency_threshold_ms：有效轮次时延阈值
- rtf_threshold：有效轮次 RTF 阈值
- max_warning_count：日志告警数量阈值
- must_contain：日志必须包含的关键片段
- must_not_contain：日志禁止出现的关键片段
- notes：报告备注

## 4. 常用命令

执行完整测试计划：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run
```

只跑时延类用例：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run --category latency
```

只跑带某个标签的用例：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run --tag stability
```

执行前先预览筛选结果：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run --tag latency --dry-run
```

保留所有音频证据：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run --download-wavs
```

说明：

- 只有在执行阶段启用 --download-wavs，report.html 才会内嵌本地音频播放器。

从已有结果重建报告：

```powershell
uv run python scripts/testing/run_tts_test_suite.py report --results-json artifacts/test-runs/<run>/results.json
```

板端上传后的运行包自带 profile 工具：

- tools/board_profile_capabilities.sh：检查 perf、strace、rknpu/load 等能力是否可用
- tools/profile_tts_inference.sh：包装 run_tts.sh，额外采集进程 RSS、线程数、utime/stime 与 rknpu load 采样

手工查看板端 profile 能力：

```bash
cd /root/tts/paddlespeech_tts_armlinux_runtime
./tools/board_profile_capabilities.sh
```

手工采集单句 profile：

```bash
cd /root/tts/paddlespeech_tts_armlinux_runtime
./tools/profile_tts_inference.sh \
	--sentence "短波电台指令测试。" \
	--output_wav ./output/profile.wav \
	--log ./output/profile.log \
	--samples-csv ./output/profile-samples.csv
```

## 5. 报告产物

默认输出目录：

- artifacts/test-runs/<suite>-<timestamp>/

目录内容：

- results.json：完整结构化结果
- results.csv：逐轮明细
- summary.md：适合评审和归档的摘要
- report.html：可直接打开的可视化仪表盘，包含输入文字和本地音频预览
- logs/：每轮执行的原始日志
- audio/：启用 --download-wavs 后回传的音频文件

## 6. 建议落地方式

按阶段拆分测试：

1. 日常联调只保留 smoke 和 baseline latency
2. 每次版本冻结前补 domain 和 acceptance
3. 发布前持续跑 stability，并保留 HTML 报告与 CSV 明细

按验收指标拆分阈值：

1. 基线阈值用于保证当前 CPU 方案不退化
2. 验收阈值用于对标最终 NPU 指标，例如 200 ms 目标
3. 两类阈值不要混在同一解释口径中
