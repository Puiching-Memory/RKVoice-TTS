# 回退手册

## 1. 触发条件

- 新运行包在板端无法启动
- 新运行包启动后无法生成有效音频
- 新运行包引入明显时延回退或稳定性问题

## 2. 回退前准备

1. 保留当前运行包压缩包
2. 保留当前 smoke_test.log 和样例音频
3. 记录当前板端部署目录与版本说明

## 3. 推荐回退策略

### 策略 A：恢复上一版运行包

1. 在 PC 保留上一版运行包压缩包
2. 将上一版运行包重新上传到板端
3. 覆盖 /root/tts/paddlespeech_tts_armlinux_runtime
4. 重新执行一次冒烟测试

### 策略 B：仅回退板端目录

1. 备份当前板端目录
2. 删除当前板端运行目录
3. 解压上一版 tar.gz 到原目录
4. 校验二进制和模型是否完整

## 4. 回退后验证

1. 再次执行冒烟测试
2. 检查是否重新生成 wav
3. 检查 smoke_test.log 中是否无异常退出
4. 人工试听确认音质无明显退化

## 5. 必须保留的回退证据

- 回退前日志
- 回退后日志
- 回退前音频样例
- 回退后音频样例
- 当前与上一版运行包版本标识
