# 发布手册

## 1. 目标

将当前工程导出为一份可交付、可归档、可传输的正式发布包。

## 2. 发布脚本

直接入口：

```powershell
./scripts/release/package_release.ps1 -Version v1.0.0
```

实际实现：

- scripts/release/package_release.ps1

说明：

- Python 相关工具推荐通过 uv run python ... 调用。
- 发布打包脚本直接使用 scripts/release/package_release.ps1。

常用参数：

- Version：发布版本号，会进入目录名、zip 名和 RELEASE_MANIFEST.md
- PackageName：发布包名前缀，默认 rk3588-tts-delivery
- ReleaseNotesPath：自定义发布说明模板或本次发布说明文件
- IncludeRuntimeBundle：附带已构建运行包压缩包
- IncludeEvidence：附带冒烟日志和音频证据

## 3. 默认发布内容

- README.md
- pyproject.toml
- uv.lock
- config/examples/
- docs/
- scripts/
- RELEASE_MANIFEST.md
- RELEASE_NOTES.md

默认不包含：

- config/local/
- .venv/
- 缓存目录
- 板端敏感配置

## 4. 可选内容

包含运行包：

```powershell
./scripts/release/package_release.ps1 -Version v1.0.0 -IncludeRuntimeBundle
```

包含验收证据：

```powershell
./scripts/release/package_release.ps1 -Version v1.0.0 -IncludeEvidence
```

全部包含：

```powershell
./scripts/release/package_release.ps1 -Version v1.0.0 -IncludeRuntimeBundle -IncludeEvidence
```

使用自定义发布说明：

```powershell
./scripts/release/package_release.ps1 -Version v1.0.0 -ReleaseNotesPath .\docs\delivery\发布说明模板.md -IncludeRuntimeBundle
```

## 5. 输出位置

默认输出目录：

- artifacts/releases/

输出内容包括：

- 一个按 包名-版本-时间戳 命名的发布目录
- 一个同名 zip 压缩包

如果未显式传 Version，则会自动使用 snapshot 版本标签。

## 6. 发布说明

默认模板：

- docs/delivery/发布说明模板.md

脚本会在发布包根目录生成 RELEASE_NOTES.md，并自动替换以下占位符：

- {{PACKAGE_NAME}}
- {{VERSION}}
- {{BUILD_TIMESTAMP}}
- {{GENERATED_AT}}
- {{RELEASE_DIRECTORY}}
- {{ZIP_PATH}}

## 7. 发布前检查

1. 确认 config/local/ 中的敏感配置不会被复制进发布包。
2. 确认 pyproject.toml 与 uv.lock 已反映当前依赖状态。
3. 确认 docs/delivery/ 中的部署、验收、回退文档与当前工程一致。
4. 如需附带运行包，确认 artifacts/runtime/paddlespeech_tts_armlinux_runtime.tar.gz 已生成。
5. 如需附带验收证据，确认 output 目录中的 wav 与 log 有效。

## 8. 发布后检查

1. 打开 RELEASE_MANIFEST.md，确认包含项正确。
2. 打开 RELEASE_NOTES.md，确认版本号、时间戳和交付范围正确。
3. 解压 zip 包，确认目录结构完整。
4. 抽检 scripts/ 目录脚本是否存在。
