# 发布手册

## 1. 目标

将当前工程导出为一份可交付、可归档、可传输的正式发布包。

## 2. 发布脚本

直接入口：

```powershell
uv run python scripts/release/package_release.py --version v1.0.0
```

容器化入口：

```powershell
uv run python scripts/release/package_release_in_docker.py --version v1.0.0
```

实际实现：

- scripts/release/package_release.py
- scripts/release/package_release_in_docker.py
- docker/package-release/Dockerfile

说明：

- Python 相关工具推荐通过 uv run python ... 调用。
- scripts/release/package_release.py 为正式入口。
- 如需把打包流程托管到容器执行，使用 scripts/release/package_release_in_docker.py；它会先构建一个最小 Python 3.12 镜像，再把当前工作区挂载到 /workspace 并在容器内执行正式入口。
- 如需附带运行包压缩包，若 workspace 中尚未存在对应 tar.gz，脚本会直接从 artifacts/runtime 下的运行包目录生成归档并写入发布包。

常用参数：

- Version：发布版本号，会进入目录名、zip 名和 RELEASE_MANIFEST.md
- PackageName：发布包名前缀，默认 rk3588-asr-tts-delivery
- ReleaseNotesPath：自定义发布说明模板或本次发布说明文件
- IncludeRuntimeBundle：附带 sherpa-onnx 运行包压缩包
- IncludeEvidence：附带 sherpa-onnx 冒烟日志和音频证据
- IncludeMeloRuntimeBundle：附带 MeloTTS-RKNN2 运行包压缩包
- IncludeMeloEvidence：附带 MeloTTS-RKNN2 冒烟日志和音频证据

## 3. 默认发布内容

- README.md
- pyproject.toml
- uv.lock
- config/examples/
- docs/
- scripts/
- RELEASE_MANIFEST.md
- RELEASE_NOTES.md

默认不包含：

- config/local/
- .venv/
- 缓存目录
- 板端敏感配置

## 4. 可选内容

包含运行包：

```powershell
uv run python scripts/release/package_release.py --version v1.0.0 --include-runtime-bundle
```

包含 MeloTTS 运行包：

```powershell
uv run python scripts/release/package_release.py --version v1.0.0 --include-melo-runtime-bundle
```

包含验收证据：

```powershell
uv run python scripts/release/package_release.py --version v1.0.0 --include-evidence
```

全部包含：

```powershell
uv run python scripts/release/package_release.py --version v1.0.0 --include-runtime-bundle --include-evidence --include-melo-runtime-bundle --include-melo-evidence
```

容器化全部包含：

```powershell
uv run python scripts/release/package_release_in_docker.py --version v1.0.0 --include-runtime-bundle --include-evidence --include-melo-runtime-bundle --include-melo-evidence
```

使用自定义发布说明：

```powershell
uv run python scripts/release/package_release.py --version v1.0.0 --release-notes-path .\docs\delivery\发布说明模板.md --include-runtime-bundle
```

## 5. 输出位置

默认输出目录：

- artifacts/releases/

输出内容包括：

- 一个按 包名-版本-时间戳 命名的发布目录
- 一个同名 zip 压缩包

如果未显式传 Version，则会自动使用 snapshot 版本标签。

## 6. 发布说明

默认模板：

- docs/delivery/发布说明模板.md

脚本会在发布包根目录生成 RELEASE_NOTES.md，并自动替换以下占位符：

- {{PACKAGE_NAME}}
- {{VERSION}}
- {{BUILD_TIMESTAMP}}
- {{GENERATED_AT}}
- {{RELEASE_DIRECTORY}}
- {{ZIP_PATH}}

## 7. 发布前检查

1. 确认 config/local/ 中的敏感配置不会被复制进发布包。
2. 确认 pyproject.toml 与 uv.lock 已反映当前依赖状态。
3. 确认 docs/delivery/ 中的部署、验收、回退文档与当前工程一致。
4. 如需附带 sherpa-onnx 运行包或证据，确认 artifacts/runtime/sherpa_onnx_rk3588_runtime/ 或其 output 目录有效。
5. 如需附带 MeloTTS 运行包或证据，确认 artifacts/runtime/melotts_rknn2_runtime/ 或其 output 目录有效。

## 8. 发布后检查

1. 打开 RELEASE_MANIFEST.md，确认包含项正确。
2. 打开 RELEASE_NOTES.md，确认版本号、时间戳和交付范围正确。
3. 解压 zip 包，确认目录结构完整。
4. 抽检 scripts/ 目录脚本是否存在。
