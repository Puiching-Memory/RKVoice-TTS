# 验收手册

## 1. 验收输入

- 项目指标：docs/requirements/项目指标.md
- 指标差距：docs/requirements/项目指标差距清单.md
- 当前运行包：artifacts/runtime/paddlespeech_tts_armlinux_runtime/

## 2. 当前仓库直接验收范围

1. 能在开发 PC 上完成 download、build、upload、all 基本流程。
2. 能在 RK3588 板端完成纯离线 TTS 合成。
3. 能将冒烟音频和测试日志自动回传到 PC。
4. 能通过 uv run python -m tests 运行当前单元测试。

当前不在本仓库直接验收范围内的内容：

- ASR 流式识别功能与指标。
- 语音→文字→语音闭环延迟与系统级资源指标。
- 普通话与主流方言混合语音交互验收。

## 3. 验收证据位置

- 冒烟日志：artifacts/runtime/paddlespeech_tts_armlinux_runtime/output/smoke_test.log
- 冒烟音频：artifacts/runtime/paddlespeech_tts_armlinux_runtime/output/smoke_test.wav
- 运行包目录：artifacts/runtime/paddlespeech_tts_armlinux_runtime/
- 单元测试入口：uv run python -m tests

## 4. 当前推荐验收步骤

1. 执行 uv run python -m tests，确认当前单元测试通过。
2. 执行 uv run python -m scripts.delivery.paddlespeech_tts_armlinux build。
3. 执行 uv run python -m scripts.delivery.paddlespeech_tts_armlinux all，完成上板和单句冒烟。
4. 检查 smoke_test.log、smoke_test.wav 和运行包目录是否在 artifacts/runtime/paddlespeech_tts_armlinux_runtime/ 下生成。
5. 人工试听关键用例音频，核对当前基线的时延、RTF、日志和运行模式。

## 5. 形成正式验收结论前还需要的补充项

1. TTS NPU 路径证明与 150 ms 指标报告。
2. 普通话、通信指令、数字字母和方言 TTS 质量评测报告。
3. TTS 内存测量报告与 7×24 稳定性报告。
4. ASR 基线实现、200 ms 流式识别报告与普通话/方言准确率报告。
5. 闭环 350 ms 延迟、总存储 ≤ 200 MB、典型工况内存 ≤ 500 MB 的系统级报告。
6. 低码率语音压缩模块对接与短波电台音频适配报告。
