# 验收手册

## 1. 验收输入

- 项目指标：docs/requirements/项目指标.md
- 指标差距：docs/requirements/项目指标差距清单.md
- ASR 运行包：artifacts/runtime/sherpa_onnx_rk3588_runtime/
- TTS 运行包：artifacts/runtime/melotts_rknn2_runtime/

## 2. 当前仓库直接验收范围

1. 能在开发 PC 上完成 ASR（sherpa-onnx）和 TTS（MeloTTS-RKNN2）的 download、build、upload、all 基本流程。
2. 能在 RK3588 板端完成纯离线 ASR 识别（CPU/ONNX + RKNN/NPU + 流式）。
3. 能在 RK3588 板端完成纯离线 TTS 合成（MeloTTS-RKNN2）。
4. 能将冒烟日志和测试结果自动回传到 PC。
5. 能通过 uv run python -m tests 运行当前单元测试。

当前不在本仓库直接验收范围内的内容：

- 语音→文字→语音闭环延迟与系统级资源指标。
- 普通话与主流方言混合语音交互验收。
- 带标注评测集的准确率评分。

## 3. 验收证据位置

- ASR 冒烟日志：artifacts/runtime/sherpa_onnx_rk3588_runtime/output/smoke_test_summary.log
- TTS 冒烟音频：artifacts/runtime/melotts_rknn2_runtime/output/smoke_test_tts.wav
- ASR 运行包目录：artifacts/runtime/sherpa_onnx_rk3588_runtime/
- TTS 运行包目录：artifacts/runtime/melotts_rknn2_runtime/
- 单元测试入口：uv run python -m tests

## 4. 当前推荐验收步骤

1. 执行 uv run python -m tests，确认当前单元测试通过。
2. 执行 uv run python -m scripts.delivery asr all，完成 ASR 上板和冒烟。
3. 执行 uv run python -m scripts.delivery tts all，完成 TTS 上板和冒烟。
4. 检查 ASR 冒烟日志和 TTS 冒烟音频是否在对应 output/ 下生成。
5. 人工试听关键用例音频，核对当前基线的时延、RTF、日志和运行模式。

## 5. 形成正式验收结论前还需要的补充项

1. TTS NPU 路径证明与 150 ms 指标报告。
2. 普通话、通信指令、数字字母和方言 TTS 质量评测报告。
3. TTS 内存测量报告与 7×24 稳定性报告。
4. ASR 200 ms 流式识别报告与普通话/方言准确率报告。
5. 闭环 350 ms 延迟、总存储 ≤ 200 MB、典型工况内存 ≤ 500 MB 的系统级报告。
6. 低码率语音压缩模块对接与短波电台音频适配报告。
