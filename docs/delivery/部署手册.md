# 部署手册

## 1. 前提条件

- Windows 开发 PC
- Docker Desktop 可用
- 已安装 uv
- 已在仓库根目录执行 uv sync
- 开发板已上电并可通过网线直连
- 本地已准备 config/local/board.local.env

## 2. 配置准备

建议优先使用：

- config/local/board.local.env
- config/local/delivery.local.env

最小必填项：

- TTS_BOARD_HOST
- TTS_BOARD_USERNAME
- TTS_BOARD_PASSWORD

可选项：

- TTS_SOURCE_IP
- TTS_REMOTE_DIR
- TTS_DOCKER_IMAGE
- TTS_DOCKER_PLATFORM
- TTS_SENTENCE

## 3. 典型流程

首次拉取项目或更新锁文件后，先同步一次本地环境：

```powershell
uv sync
```

后续 Python 工具统一通过 uv run 执行：

### 3.1 首次准备离线源码包

```powershell
uv run python scripts/delivery/prepare_paddlespeech_tts_armlinux.py download
```

### 3.2 本地构建 ARM64 运行包

```powershell
uv run python scripts/delivery/prepare_paddlespeech_tts_armlinux.py build
```

说明：

- 第一次会创建本地 Docker builder 镜像。
- 后续构建会复用该镜像，不再重复安装系统依赖。

### 3.3 上传到开发板并执行冒烟测试

```powershell
uv run python scripts/delivery/prepare_paddlespeech_tts_armlinux.py upload
```

如果是链路本地网段且需要显式绑定源地址：

```powershell
uv run python scripts/delivery/prepare_paddlespeech_tts_armlinux.py upload --source-ip 169.254.46.223
```

### 3.4 一次完成构建、上传和测试

```powershell
uv run python scripts/delivery/prepare_paddlespeech_tts_armlinux.py all
```

### 3.5 执行批量测试并生成可视化报告

```powershell
uv run python scripts/testing/run_tts_test_suite.py run
```

只跑时延类用例：

```powershell
uv run python scripts/testing/run_tts_test_suite.py run --category latency
```

## 4. 结果检查

成功后应检查：

- 本地运行包目录：artifacts/runtime/paddlespeech_tts_armlinux_runtime/
- 本地音频回传：artifacts/runtime/paddlespeech_tts_armlinux_runtime/output/smoke_test.wav
- 本地测试日志：artifacts/runtime/paddlespeech_tts_armlinux_runtime/output/smoke_test.log
- 批量测试 HTML 报告：artifacts/test-runs/<suite>-<timestamp>/report.html
- 批量测试 CSV 明细：artifacts/test-runs/<suite>-<timestamp>/results.csv

## 5. 板端目录

默认上传目录：

- /root/tts/paddlespeech_tts_armlinux_runtime

## 6. 常见问题

1. 如果 upload 失败，先检查 board.local.env 中的主机、用户名、密码和源地址是否正确。
2. 如果构建失败，先检查 Docker Desktop 是否可用，以及本地 builder 镜像是否已创建。
3. 如果测试句子包含英文词导致前端异常，先改用纯中文句子做冒烟验证。
