# 部署手册

## 1. 前提条件

- Windows 开发 PC
- Docker Desktop 可用
- 已安装 uv
- 已在仓库根目录执行 uv sync
- 开发板已上电并可通过网线直连
- 本地已准备 config/local/board.local.env

## 2. 配置准备

建议优先使用：

- config/local/board.local.env
- config/local/delivery.local.env

最小必填项：

- TTS_BOARD_HOST
- TTS_BOARD_USERNAME
- TTS_BOARD_PASSWORD

可选项：

- TTS_SOURCE_IP
- TTS_REMOTE_DIR
- TTS_DOCKER_IMAGE
- TTS_DOCKER_PLATFORM
- TTS_SENTENCE

## 3. 典型流程

首次拉取项目或更新锁文件后，先同步一次本地环境：

```powershell
uv sync
```

后续 Python 工具统一通过 uv run 执行：

### 3.1 首次准备离线源码包

```powershell
uv run python -m scripts.delivery.paddlespeech_tts_armlinux download
```

### 3.2 本地构建 ARM64 运行包

```powershell
uv run python -m scripts.delivery.paddlespeech_tts_armlinux build
```

说明：

- 第一次会创建本地 Docker builder 镜像。
- 后续构建会复用该镜像，不再重复安装系统依赖。

### 3.3 上传到开发板并执行冒烟测试

```powershell
uv run python -m scripts.delivery.paddlespeech_tts_armlinux upload
```

如果是链路本地网段且需要显式绑定源地址：

```powershell
uv run python -m scripts.delivery.paddlespeech_tts_armlinux upload --source-ip 169.254.46.223
```

### 3.4 一次完成构建、上传和测试

```powershell
uv run python -m scripts.delivery.paddlespeech_tts_armlinux all
```

### 3.5 执行当前可用验证

```powershell
uv run python -m tests
```

说明：

- 当前仓库已提供单元测试、板端构建、上板部署和单句冒烟能力。
- 当前仓库未提供独立的板端批量 TTS 验收 CLI，也未提供 ASR 与闭环系统测试入口。
- 如果要对标最终项目指标，当前建议以 build / upload / all 命令的冒烟结果作为 TTS 基线证据，再单独补充 NPU、ASR 和闭环测试方案。

## 4. 结果检查

成功后应检查：

- 本地运行包目录：artifacts/runtime/paddlespeech_tts_armlinux_runtime/
- 本地音频回传：artifacts/runtime/paddlespeech_tts_armlinux_runtime/output/smoke_test.wav
- 本地测试日志：artifacts/runtime/paddlespeech_tts_armlinux_runtime/output/smoke_test.log
- 单元测试入口：uv run python -m tests
- 板端 profile 工具：artifacts/runtime/paddlespeech_tts_armlinux_runtime/tools/

## 5. 板端目录

默认上传目录：

- /root/tts/paddlespeech_tts_armlinux_runtime

## 6. 常见问题

1. 如果 upload 失败，先检查 board.local.env 中的主机、用户名、密码和源地址是否正确。
2. 如果构建失败，先检查 Docker Desktop 是否可用，以及本地 builder 镜像是否已创建。
3. 如果测试句子包含英文词导致前端异常，先改用纯中文句子做冒烟验证。
4. 当前 smoke test 只能证明 TTS 基线可运行，不能直接替代最终 150 ms TTS 指标、200 ms ASR 指标或 350 ms 闭环指标验收。
