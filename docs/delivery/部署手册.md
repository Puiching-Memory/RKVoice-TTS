# 部署手册

## 1. 前提条件

- Windows 开发 PC
- Docker Desktop 可用
- 已安装 uv
- 已在仓库根目录执行 uv sync
- 开发板已上电并可通过网线直连
- 本地已准备 config/local/board.local.env

## 2. 配置准备

建议优先使用：

- config/local/board.local.env
- config/local/delivery.local.env

最小必填项：

- TTS_BOARD_HOST
- TTS_BOARD_USERNAME
- TTS_BOARD_PASSWORD

## 3. 典型流程

首次拉取项目或更新锁文件后，先同步一次本地环境：

```powershell
uv sync
```

后续 Python 工具统一通过 uv run 执行：

### 3.1 ASR 运行包（sherpa-onnx）

#### 首次准备离线源码包

```powershell
uv run python -m scripts.delivery asr download
```

#### 本地组装运行包

```powershell
uv run python -m scripts.delivery asr build
```

#### 上传到开发板并执行冒烟测试

```powershell
uv run python -m scripts.delivery asr upload --source-ip 169.254.46.223
```

#### 一次完成组装、上传和测试

```powershell
uv run python -m scripts.delivery asr all --source-ip 169.254.46.223
```

### 3.2 TTS 运行包（MeloTTS-RKNN2）

#### 首次准备离线源码包

```powershell
uv run python -m scripts.delivery tts download
```

#### 本地组装运行包

```powershell
uv run python -m scripts.delivery tts build
```

#### 上传到开发板并执行冒烟测试

```powershell
uv run python -m scripts.delivery tts upload --source-ip 169.254.46.223
```

#### 一次完成组装、上传和测试

```powershell
uv run python -m scripts.delivery tts all --source-ip 169.254.46.223
```

### 3.3 执行当前可用验证

```powershell
uv run python -m tests
```

## 4. 结果检查

成功后应检查：

- ASR 运行包目录：artifacts/runtime/sherpa_onnx_rk3588_runtime/
- ASR 冒烟日志：artifacts/runtime/sherpa_onnx_rk3588_runtime/output/smoke_test_summary.log
- TTS 运行包目录：artifacts/runtime/melotts_rknn2_runtime/
- TTS 音频回传：artifacts/runtime/melotts_rknn2_runtime/output/smoke_test_tts.wav
- 单元测试入口：uv run python -m tests

## 5. 板端目录

默认上传目录：

- ASR：/root/rkvoice/sherpa_onnx_rk3588_runtime
- TTS：/root/rkvoice/melotts_rknn2_runtime

## 6. 常见问题

1. 如果 upload 失败，先检查 board.local.env 中的主机、用户名、密码和源地址是否正确。
2. 如果测试句子包含英文词导致前端异常，先改用纯中文句子做冒烟验证。
3. 当前 smoke test 只能证明基线可运行，不能直接替代最终 150 ms TTS 指标、200 ms ASR 指标或 350 ms 闭环指标验收。
